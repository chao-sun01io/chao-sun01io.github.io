行动是解决 AI 焦虑的良药

Chao
 2026-04-14  约 51 字   预计阅读 1 分钟 
目录

Token 诚可贵，行动价更高

不知道读者朋友们有没有因为AI的快速发展而感到焦虑？ 我对AI的感受是经历了一个从怀疑、好用、惊艳、焦虑的过程。 AI已经是房间里的大象, 逐步的成为我们生活的一部分，我们可能主动拥抱它也有可能会选择视而不见，但是这都不会影响它的发展一日千里，这么一想我更焦虑了。

我算是比较晚相信AI，并且应用AI的人了，八九年前我知道深度学习能够很好地解决一些现实中的问题，特别是在语音和图像领域，但是我万万没有想到他能够达到今天如此先进的水平。 在过去的好几年里，我一直把Chatbot当作一个搜索引擎的进化版本, 其实这个概念早在2015年的时候我就听过，当时在清华， 吴军博士来做访谈讲座，他就提到过未来搜索引擎的方向是你给出问题他会给你答案，而不是给你链接，今天真的已经完全实现了。

2022年前 Chatgpt 横空出世， 最开始只是以为是个又一个 Demo, 没想到它是如此的惊艳，很快同类的Chatbot 百花齐放, 替代了传统搜索引擎，作为我的最主要的查询入口。后面各种 Agent 也发展起来了，以我所在的编程领域为例，从最开始的代码补全到后面模块级别代码生成以及到现在的软件生成，各种新术语如 Vibe coding、Agentic Engineering, MCP， OpenClaw(龙虾)， Harness, 各种新的名词，新的技术层出不穷，AI发展比我学得还快，能不焦虑吗？

一个已经被公认的事实就是AI正在给产业和社会带来深刻变革，我们在其中需要找到自己的位置，不能蒙住眼睛，等待时代的浪潮打在头上，就算我们无法在商业上参与AI获得机会，我们也应当主动拥抱AI技术， 去发挥自己的更大潜力。最近几个月我开始更深度地应用AI，用 Claude Code 协作来做个人项目， 效果特别好，很快做了个展示基金K线图的网站，并且迅速上线了。 这个过程中，我除了惊叹AI编程的强大之处，也发现自己作为有经验的程序员也是有优势的，我可以把需求拆分成粒度合适、层层递进的用户故事，让生成的程序克制而满足需求；我能够根据自己经验做减法，限制AI过度生成代码； 我能够在AI生成的部署代码出错时，用古法编程的方式进行调试； AI也能弥补我在一些技术栈的经验不足，比如前端和样式设计。

经历这个过程， 我反而没有那么焦虑了， AI对程序员乃至所有工作来说，都是一种加强。 对于有很多想法的人来说，有了AI他们如鱼得水，可以快速的将想法变成现实。我想人类是要与AI共生的， 神经可塑性不仅是AI的优势，也是我们人类（如果要非要加一个曾经的）优势，我们不需想着与AI在人类弱势的领域竞争（例如记忆，计算），君子善假于物，我们可能以某种方式把AI的能力和我们自己的个体的优势嵌合到一起， 让个体能够发挥出更大的潜力。 以下是我对于如何缓解AI焦虑的一些想法：

  1. 结合自己的个性和独特经历，找到自己的相对AI和其他人的优势，并且加强这个优势
  2. AI 虽然发展迅速，但是目前的物理世界终究是人类活动主导的，拥有资产和承担责任仍然是人类和类似公司这样的法律实体的特殊能力
  3. 活在当下，专注于当前的工作和生活，能够提升幸福的体验
  4. 采取行动，使用AI，也许是从写出更好的提示词开始获得更加精准的答案，也许是从应用AI工具解决一下具体的现实问题开始：做一个App，自动化表格等等，把想做的事情用AI给实现出来。亲身把事情做出来做成功，这个过程中也会提升人的自我效能感——心理学术语，个体对自己是否有能力完成某一行为所进行的推测与判断， 通俗来讲，觉得自己又行了。

接下来的几期准备连载几期自己探索使用AI的经历和心得体会， 欢迎大家探讨。
